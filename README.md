# 惠州仲恺中学 - Hexo 文章自动处理工作流

一个基于 GitHub Actions 的自动化工作流，使用 AI 将微信公众号文章批量转换为规范的 Hexo 博客文章格式，并通过 **Pull Request** 提交供人工审核。

## ✨ 特性

- 🤖 **AI 智能处理** - 支持 Gemini / OpenAI，自动转换文章格式、生成元信息
- 📅 **智能排序** - 从文件名提取日期，优先处理新文章
- 🖼️ **图片自动迁移** - AI 处理后将微信图片迁移到你的图床（WebP 格式）
- 📝 **PR 审核模式** - 处理完成后创建 PR，供人工审核后合并
- 📊 **进度追踪** - 完整的状态管理，支持 1000+ 文章的断点续处理
- 🔄 **每日限额** - 控制每天处理的文章数量，避免 API 超支
- ✅ **质量检查** - 自动验证 Front Matter、分类体系、内容结构
- 🏫 **学校定制** - 针对惠州仲恺中学的 SEO 优化和分类体系

## 🚀 快速开始

### 步骤 1: 准备文章文件

将你的原始文章放入 `raw-articles/` 目录，支持以下文件名格式：

```
[2026-01-26]聚焦智慧教育赋能精准教学.md    ✅ 推荐
2026-01-26-聚焦智慧教育赋能精准教学.md      ✅ 支持
20260126聚焦智慧教育赋能精准教学.md          ✅ 支持
任意文件名.md                               ⚠️ 会排在最后处理
```

**注意**：系统会自动从文件名提取日期，**优先处理新文章**。

### 步骤 2: 配置 Secrets

在 GitHub 仓库设置中添加以下 Secrets：

| Secret 名称 | 说明 | 必需 |
|------------|------|------|
| `GEMINI_API_KEY` | Google Gemini API 密钥 | ✅ (使用 Gemini) |
| `OPENAI_API_KEY` | OpenAI API 密钥 | ✅ (使用 OpenAI) |
| `HEXO_REPO` | Hexo 仓库地址（格式：username/repo） | ✅ |
| `HEXO_REPO_TOKEN` | 有推送权限的 Personal Access Token | ✅ |
| `IMAGE_API_TOKEN` | 图床 API Token | ✅ |
| `IMAGE_HOST_URL` | 图床地址（默认：https://photo.20080601.xyz） | ❌ |
| `IMAGE_DISPLAY_URL` | 图片显示域名（默认：https://photo1.20080601.xyz） | ❌ |
| `PR_REVIEWERS` | PR 审核者（逗号分隔的 GitHub 用户名） | ❌ |

### 步骤 3: 自定义提示词（可选）

编辑 `prompts/transform.txt` 文件，根据你的需求调整 AI 处理提示词。

### 步骤 4: 运行工作流

**自动运行：**
- 每天 UTC 00:00（北京时间 08:00）自动执行

**手动运行：**
1. 进入 Actions 页面
2. 选择 "Hexo Article Processor"
3. 点击 "Run workflow"
4. 可选参数：
   - **处理数量**：本次处理几篇文章
   - **强制重新处理**：是否重新处理已完成的文章
   - **AI 提供商**：选择 Gemini 或 OpenAI
   - **跳过图片迁移**：是否跳过图片迁移步骤

## 🖼️ 图片迁移功能

### 工作原理

1. **AI 处理完成后**才运行图片迁移，避免处理被删除的内容浪费图床空间
2. 自动下载微信图片
3. 转换为 WebP 格式（节省 30-70% 空间）
4. 上传到你的图床
5. 替换文章中的图片链接

### 缓存机制

- 已上传的图片 URL 会缓存在 `image_migration_cache.json`
- 相同图片不会重复上传
- 缓存文件会自动提交到仓库

### 配置参数

| 环境变量 | 说明 | 默认值 |
|---------|------|--------|
| `IMAGE_API_TOKEN` | 图床 API Token | - |
| `IMAGE_HOST_URL` | 图床上传地址 | https://photo.20080601.xyz |
| `IMAGE_DISPLAY_URL` | 图片显示域名 | https://photo1.20080601.xyz |
| `IMAGE_UPLOAD_CHANNEL` | 上传通道 | telegram |
| `IMAGE_TARGET_FOLDER` | 目标文件夹 | wx |

## ⚠️ 注意事项

1. **API 费用**：每天处理的文章数量会消耗 API 额度，建议设置合理的 `DAILY_LIMIT`

2. **图片迁移顺序**：图片迁移在 AI 处理**之后**运行，这样：
   - AI 可能删除不需要的图片（如装饰图）
   - 只迁移最终保留的图片，节省图床空间

3. **人工审核**：PR 创建后请务必人工审核内容质量，特别是：
   - 标题是否准确、有吸引力
   - 分类和标签是否合适
   - 图片链接是否有效
   - 内容格式是否正确

4. **失败重试**：失败的文章会自动重试，最多 3 次

5. **状态文件**：`status.json` 和 `image_migration_cache.json` 会自动提交到仓库，请勿手动删除

## 🔧 常见问题

### Q: 为什么新文章优先处理？

A: 系统从文件名提取日期（如 `[2026-01-26]`），按日期降序排序，确保最新的内容优先发布。

### Q: 如何切换 AI 提供商？

A: 在手动触发工作流时选择 AI 提供商，或修改 `DEFAULT_AI_PROVIDER` 环境变量。

### Q: 图片迁移失败怎么办？

A: 
- 检查 `IMAGE_API_TOKEN` 是否正确
- 查看 `image_migration_failed.json` 了解失败详情
- 失败的图片会保留原始微信链接

### Q: 如何跳过图片迁移？

A: 手动触发时勾选 "跳过图片迁移" 选项。

### Q: 如何重新处理某篇文章？

A: 修改 `status.json` 中对应文章的状态为 `pending`，并将 `attempts` 重置为 0。
